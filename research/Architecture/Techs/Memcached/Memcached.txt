
1. 什么是Memcached

免费开源、高性能、分布式的内存对象缓存系统，实际上具有通用性，但其目的用于通过减轻数据库负担来加速动态Web应用。

Memcached是一个驻内存的key-value存储，用于(存储)来自数据库调用、API调用、或页面渲染结果的小块任意数据(字符串，对象)。

Memcached简单却强大，它的简单设计加速了快速部署，轻松开发，并解决了面对大数据缓存的很多问题。它的API对大多数流行语言都是可用的。

你可以把它想成你应用的一个短期内存。

2. Memcached做什么

memcache让你可以从你系统的部分拿走内存，那里内存多于需求，并且使之可用于持有内存少于需求的地方。

memcache也让你可以更好的利用内存，考虑右边的图，你能看到两种部署场景：

(1) 每个节点完全独立(上)
(2) 每个节点可以利用其他节点的内存(下)

第一个场景说明了典型的部署模式，但是你会发现它很浪费，不仅在于总的缓存大小是你web框架上实际可用容量的一个分数，而且在于要求保持节点间缓存一致性的总花费上。

使用memcache，你可以看到所有服务器看到的都是相同的虚拟内存池。这意味着在整个web集群中，给定的(数据)项总是存储并且获取于同样的位置。

同样，当你的应用增长到需要更多服务器的时候，通常需被经常访问的数据同样也增长了。一个能使你的系统这两方面同时扩张的部署策略是有意义的。

为简便，右边的图示仅显示两个web服务器，但随着数目的增加性质是一样的。如果你有50台web服务器，在第一个例子中你仍然只有64M可用缓存，但第二例子中，你将有3.2G可用缓存。

当然，你并不被要求使用你web服务器的内存作为缓存。很多memcached使用者有专用的机器，专门建来只用作memcached服务器。

5. 安装

Debian/Ubuntu: apt-get install libevent-dev 
Redhat/Centos: yum install libevent-devel 

wget http://memcached.org/latest
tar -zxvf memcached-1.x.x.tar.gz
cd memcached-1.x.x
./configure && make && make test && sudo make install
