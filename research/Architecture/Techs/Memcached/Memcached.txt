关于Memcached

1. Memcached是什么

免费开源、高性能、分布式的内存对象缓存系统，有一般的通用性，但其目的在于通过减轻数据库负担来加速动态Web应用。

Memcached是一个驻内存的key-value存储，用于存储小块的任意数据(字符串，对象)，这些数据来自数据库调用、API调用、或页面渲染的结果。

Memcached简单却强大，它的简单设计提升了快速部署、易于开发、并解决了面对大数据缓存的很多问题。它的API对大多数的流行语言都是可用的。

你可以把它想象成你应用的一个短期内存。

2. Memcached做什么

memcached让你可以从持有多于需求的系统部分拿走内存，并且使之可被持有少于需求的部分使用。

memcached也让你可以更好的利用内存。如果你考虑右(下)边的图，你能看到两种部署场景：


(1) 每个节点完全独立(上)
(2) 每个节点可以利用其他节点的内存(下)

第一个场景说明了典型的部署策略，但是你会发现它很浪费，不仅在于总的缓存大小是你web框架上实际可用容量的一个分数，而且在于对保持节点间缓存一致性需求的开销上。

使用memcached，你可以看到所有服务器查找的都是同一个虚拟内存池。这意味着在整个web集群中，给定的(数据)项总是存储并且获取于同样的位置。
同样，当你的应用增长到需要更多服务器的时候，需要被经常访问的数据通常也增长了。一个能使你的系统这两方面同时扩张的部署策略是有价值的。
为简便右边的图示仅展示两个web服务器，但随着数目的增加性质是一样的。如果你有50台web服务器，在第一个例子中你仍然只有64M可用缓存，但第二例子中，你将有3.2G可用缓存。

当然，你并不一定要使用你的web服务器的内存作为缓存。很多memcached使用者有专门的机器，建来只用作memcached服务器。
 
3. 起源

Memcached最初是由Brad Fitzpatrick（人名）在2003年为LiveJournal开发的。

Danga Interactive（公司）开发memcached用来提升LiveJournal.com的速度，此网站平均每天有2千万多的动态页面浏览和1百万用户，并有一组web服务器和一组数据库服务器。memcached使数据库的负载下降到几乎没有，随之而来的是给用户更快的页面加载时间，更好的资源利用率，及memcached失效时更快的数据库访问。

5. 安装

Debian/Ubuntu: apt-get install libevent-dev
Redhat/Centos: yum install libevent-devel

wget http://memcached.org/latest
tar -zxvf memcached-1.x.x.tar.gz
cd memcached-1.x.x
./configure && make && maketest && sudo make install
